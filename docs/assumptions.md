# OSE 项目关键假设

## 产品与业务假设
- 系统仅服务于单一管理员用户，默认用户名为 `admin`，不开放注册。
- MVP 以桌面浏览器体验为主，移动端仅提供基础响应式，不追求复杂手势与深度适配。
- 目标考试默认为 2026 年上半年软件设计师考试，允许在系统设置中修改为任意日期。
- 学习计划采用启发式规则生成，而非复杂排程优化算法，以保证稳定和可维护。
- 知识体系预置三级结构，后续允许扩展字段但不在本期做版本化知识库。

## 技术假设
- 前后端采用单仓库前后端分离结构：`frontend/` + `backend/`。
- 后端采用 Spring Boot 3.x + Java 17 + Spring Data JPA + Flyway + MySQL。
- 前端采用 Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router。
- 图表方案默认使用 ECharts；Markdown 渲染采用成熟开源库。
- 认证采用 JWT Bearer Token，MVP 仅实现单管理员登录，不实现刷新令牌黑名单等复杂安全机制。

## 运行与部署假设
- 推荐通过 Docker Compose 启动完整环境；本地开发可分别运行前后端。
- 鉴于当前环境缺少 Maven，仓库将提供 Docker 化 Maven 执行方式与 Compose 构建路径。
- 数据初始化由数据库迁移 + 应用启动 seed 逻辑共同完成。
- 默认数据库为 MySQL 8，字符集为 `utf8mb4`。

## 数据与内容假设
- 示例数据均为自拟内容，仅用于演示软件设计师备考场景。
- 上午题采用单选模型；下午题采用主观问答模型，参考答案只提供要点摘要。
- 错题复习周期默认为 1 / 3 / 7 / 14 天递进，可在设置中修改基础间隔。

## 工程与质量假设
- 最小有效测试覆盖以下关键点：认证、计划生成、练习判分、统计聚合、前端关键工具 / Store。
- 基础 CI 以 lint / test / build 为主，不在本期引入复杂质量平台。
- 安全侧重点为输入校验、密码哈希、基础鉴权、CORS、统一异常处理和最小暴露面。
