# OSE（Open Software Exam）需求基线

## 1. 项目目标
- 构建一个单用户、中文优先、PC 优先的 Web 备考系统，服务于 2026 上半年软考（软件设计师）备考。
- 通过统一系统承载计划管理、题库管理、练习、模拟考试、错题复盘、笔记与统计分析。
- 保证系统在本地无外部 AI、无第三方密钥的条件下可完整运行与演示核心流程。

## 2. 用户画像
- 用户身份：单一管理员，即系统拥有者本人。
- 使用场景：下班后与周末在桌面浏览器进行刷题、复盘、计划编排和进度回顾。
- 核心诉求：减少分散工具切换，形成“目标→计划→练习→纠错→复盘→统计”的闭环。
- 技术预期：希望部署简单、维护成本低、首次启动顺畅。

## 3. 用户主流程
1. 首次启动系统并使用默认管理员账号登录。
2. 配置考试目标日期、通过阈值、每周学习时长与学习偏好。
3. 自动生成阶段计划、周计划与日计划，并在日常学习中更新任务状态。
4. 浏览知识树，查看各知识域掌握度与关联题目。
5. 从题库中进行按知识点练习、随机练习或错题复习。
6. 上午题自动判分，下午题保存作答、自评与复盘记录。
7. 创建并完成模拟考试，查看历次成绩与薄弱项变化。
8. 使用错题本与笔记系统持续复盘。
9. 在统计分析页查看计划完成率、正确率、模拟成绩趋势和错题分布。
10. 在系统设置页进行目标调整、数据导入导出与健康检查。

## 4. MVP 范围

### 4.1 仪表盘
- 考试目标日期展示与倒计时
- 今日学习任务
- 本周 / 本月完成情况
- 知识域掌握度概览
- 最近错题、最近模拟考、最近笔记

### 4.2 学习计划
- 配置考试日期、每周可学习时长、学习偏好
- 自动生成阶段计划（基础 / 强化刷题 / 冲刺复盘）
- 自动拆分周计划与日任务
- 支持任务状态、优先级、完成度、延期和补做

### 4.3 知识点体系
- 支持一级 / 二级 / 三级知识点
- 支持掌握度、权重、备注
- 支持关联题目与笔记

### 4.4 题库管理
- 上午选择题、下午主观题两类题型
- 支持题目录入、编辑、删除、筛选、标签
- 支持示例 CSV / JSON 导入模板
- 系统仅内置自拟示例题，不内置受版权保护的历年真题全文

### 4.5 练习系统
- 按知识点练习、随机练习、仅错题练习
- 收藏题、标记不会、加入复习
- 上午题自动判分
- 下午题支持作答、自评、参考要点与作答记录

### 4.6 模拟考试
- 创建模拟卷
- 上午卷 / 下午卷模式
- 计时、交卷、成绩记录
- 上午自动评分、下午自评复盘

### 4.7 错题本与复习
- 自动收集错题
- 错因分类、复习时间、状态流转
- 支持按知识点聚合查看

### 4.8 笔记
- Markdown 笔记的增删改查
- 关联知识点、题目、模拟考
- 收藏与搜索

### 4.9 统计分析
- 按知识点统计练习次数、正确率、趋势
- 计划完成率
- 模拟成绩趋势
- 错题分布图表

### 4.10 系统设置与数据能力
- 考试目标日期、通过阈值、默认复习周期设置
- 部分业务数据导出 / 导入
- 基础健康检查接口

## 5. 非目标
- 多用户注册、组织协作、多租户
- 支付、订阅、社区、排行榜、聊天
- 手机原生 App
- 强依赖云端 AI 的核心流程
- 爬取互联网真题或教材全文
- 复杂工作流引擎与消息推送平台

## 6. 核心功能清单
- 认证登录与默认管理员初始化
- 仪表盘数据聚合
- 学习设置与计划生成
- 知识树维护
- 题库维护与筛选
- 练习作答与自动 / 手动评分
- 错题记录与复习机制
- 模拟考试管理
- Markdown 笔记
- 统计分析图表
- 数据导入导出
- 健康检查与基础日志

## 7. 关键验收标准
- `docker compose up -d --build` 后，前端、后端、数据库均可启动。
- 使用默认管理员账号可登录系统。
- 系统自动初始化知识点、自拟示例题、示例计划、示例模拟记录与示例笔记。
- 可完成如下核心链路：
  - 配置考试日期与学习参数
  - 自动生成学习计划
  - 浏览知识点与题库
  - 完成上午题并自动判分
  - 完成下午题并保存作答与自评
  - 错题自动入库并进行复习流转
  - 创建 / 查看模拟考试记录
  - 查看统计图表与进度面板
- 仓库包含 `.env.example`、迁移脚本、README、交付报告与最小有效测试。

## 8. 风险与约束
- 仓库初始为空，需要从 0 搭建前后端与部署资产。
- 主机缺少本地 Maven / Gradle，需要优先采用 Docker 化构建与运行策略。
- 项目为个人使用，功能设计应优先覆盖主链路而非多角色扩展。
- 为控制复杂度，本期统计分析以可解释、可验证的聚合指标为主，不引入复杂预测模型。
- 为避免版权风险，题库只提供自拟示例题和导入模板。
